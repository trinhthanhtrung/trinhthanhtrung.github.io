Learn to Drive

Car driving simulation system

Learn to drive is a complete simulation system which provides users an environment similar to a car with many features in a real car. By using this simulation, users who are not familiar with driving a car will be able to learn the basic of driving as well as various on-the-road driving situation.

Main features:

  • Detailed car model. All components are designed with high precision compared to those in real car.
  • Support various driving terrains and real life roads, e.g Tran Duy Hung Str. (Hanoi, Vietnam)
  • Many driving lessons are provided, including several real-life situation and traffic laws practice.

Fuzzy Logic Implementation

A demonstration of how fuzzy logic can be implemented in a game

The main aim of this project is to research and demonstrate some artificial intelligence (AI) implementations in computer games. Two AI algorithms have been implemented, namely Fuzzy Logic and Bayesian Networks. The secondary aim of the project is to apply several concepts of game programming by creating an actual application. An actual working game demo has been created. There is a non-playable character (NPC) in the demo which is able to make various decisions with the help of AI algorithms. In order to achieve this result, the algorithms are researched thoroughly and then applied to a game demo created using a rendering engine.

Application:

The application consists of two different implementations

2. Fuzzy logic implementation

Users can choose to run the application with or without fuzzy logic implementation so that they can see the differences. The guardian with fuzzy logic implementation is initialized with the fuzzy rules and fuzzy variable. If the application is run without fuzzy logic
implementation, the guardian will attack the enemies who are nearest to the player.

2. Bayesian implementation

Users can choose to run the application with no initial evidence database, with default initial evidence database or they can create the initial evidence database themselves. In case the default initial evidence database is chosen, the database is indicated as below:

  1. In 38 chests that were small and grey, 1 contained gold
  2. In 26 chests that were small and golden, 8 contained gold
  3. In 5 chests that were big and grey, 3 contained gold
  4. In 10 chests that were big and golden, 9 contained gold
